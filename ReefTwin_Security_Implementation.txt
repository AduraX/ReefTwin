Create a security baseline for the project aligned with:
- ASD Essential Eight principles

- CIS Kubernetes Benchmark concepts
- NIST AI RMF 1.0 principles

Create:
- docs/security/security_baseline.md
- docs/security/threat_model.md

The threat model must cover:
- reef state API
- simulation API
- ingestion pipelines
- Kafka/Redpanda topics
- object storage

Use STRIDE-style categories:
- spoofing

- denial of service
- privilege escalation

Do not claim full compliance. Use the wording “security-aligned with”.

3. Role-based access control

Add RBAC authorization to ReefTwin.

Create roles:
- reef_admin
- scientist
- analyst
- public_viewer

Permissions:
- reef_admin: full access
- scientist: upload datasets, run simulations, view all reef states
- analyst: view reef states and dashboards, cannot upload or mutate data
- public_viewer: read-only access to public reef summaries

Implement:
- role extraction from JWT claims
- permission decorators/dependencies
- object-level authorization for reef_id
- deny-by-default access control

Protect endpoints:
- POST /simulate requires scientist or reef_admin
- POST /datasets/upload requires scientist or reef_admin
- GET /reefs/{reef_id}/state requires object-level permission
- GET /public/reefs requires public access

Add tests for privilege escalation and object-level access bypass.
